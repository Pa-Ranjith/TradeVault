# TradeVault - Deployment & Project Guide

Welcome to the **TradeVault** repository. This guide provides clear instructions on how this project is structured and how to keep it running "live".

## 🚀 How to Deploy (Netlify)

This project is configured to deploy automatically via **Netlify**. 

### **The Setup (Technical simplified)**:
- **Base Directory:** `frontend` (The "engine" of the app lives here).
- **Build Command:** `npm run build` (This command tells the computer to package the code for the web).
- **Publish Directory:** `.next` (This is where the final, finished website files are placed).

### **To Update the Live Site**:
1. Simply push your changes to the `main` branch on GitHub.
2. Netlify will see the change and start a "Deploy" automatically.
3. If it fails, check the `netlify.toml` file at the root—it contains the "secret sauce" instructions that tell Netlify where to look.

## 📂 Project Structure
- **/frontend**: Contains the user interface, styling, and logic (Next.js).
- **/database**: Contains data structures or local storage logic.
- **netlify.toml**: The bridge between GitHub and Netlify. **Do not delete this.**
